Ordered logistic regressions¶

Ordered logistic regressions allow us to model an outcome that is categorical but that for which the order of those categories matters. Examples include educational attainment (which we'll look at here), employment (unemployed, part-time employed, full-time employed), or, very commonly, Likert-scale survey responses.

We will use respondents' ethnicity as hispanic or latino to predict educational attainment among adults in the United States. First, load the data and create a categorical variable for educational attainment with levels of "None", "High School", and "College".

To formulate our model, we will use the dordlogit() function. This is a shortcut function that builds an ordered logit model based on two arguments:

  • The first argument (phi) is the linear model portion of the specification ($\phi$ from the slides)
  • The second argument is a vector of $\alpha_k$ values, the intercepts of the model. For a model with $k$ outcome categories, you should give dordlogit() $k-1$ $\alpha$ parameters.
In [4]:
m <- alist(
    ed_level ~ dordlogit(phi,c(a1,a2)),
    phi <- b*hispanic,
    c(a1,a2) ~ dnorm(0,1.5),
    b ~ dnorm(0,2)
)

The model can be fit using map(), but we need to be a little bit careful. The priors we set for a1 and a2 are independent, so the MAP algorithm might start its search somewhere with a2<a1, which is not allowed and has zero likelihood. To avoid this, we will give map() function specific start values for these parameters.

In [5]:
# find complete cases for our model (two variables)
cc <- complete.cases(d[c('ed_level','hispanic')])

# define start values for a1 and a2. These are pretty
# arbitrary, but a2 must be bigger than a1.
a_start <- list(a1=-1,a2=1)

# fit the model and look at the results
fit <- map(m,data=d[cc,],start=a_start)
precis(fit)

The estimates for a1 and a2 are a little hard to interpret on their own, but we could notice that they are pretty far apart from one another. This suggests that the middle category (high school) has a large overall probability in the model.

We can also see that the estimate for b is negative, which says that hispanic respondents are predicted to have less education overall.

Using the inv_logit() function, we can calculate probabilities based on these estimates.
