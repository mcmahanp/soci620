Bayesian regressions¶

In this notebook, we will learn how to estimate linear regression models in a Bayesian framework. This all may seem like a lot of extra work since lm() is fast and easy to use, but learning these techniques on a model you are familiar with will help later on when we start working with more complex models (I promise).

Let's build our first model. We'll make an alist describing a model predicting log income as normally distributed, but with a mean that depends on a person's gender.

(Note: the US census, which provided this data, does not collect information on non-binary gendereveryone is either a man or a woman. This is an unfortunate limitation of a great deal of existing survey data and is only just beginning to change. Although I recoded and formatted this data, I left the label "female" as it is in the raw data.)

In [2]:
m1 <- alist(
    log_income ~ dnorm(mu,sigma),
    mu <- a + b*female,
    a ~ dnorm(0,30),
    b ~ dnorm(0,30),
    sigma ~ dunif(0,50)
)

The big difference between this model and models we've been using so far is that mu is defined using <- instead of ~. In the rethinking package and in Stan, the "tilde" ~ is used for stochastic or random relations, while the "assignment arrow" <- is used for deterministic relations. This tells the model that log_income is considered random, even if you know mu and sigma, but that you can calculate mu exactly as long as you have speific values for a, b, and female.

We can estimate this model using map():

The precis() function will describe the marginal posterior distributions for each of our parameters, producing output similar to a regular regresssion:

In [4]:
# `prob` specifies the size of the credible interval to calculate
# `digits` speifies how many digits to show for each result
precis(fit1, prob=.95, digits=3)

We can extract the coefficients from this object to get the expected income for men and women, conditional on the data we have seen.

In [5]:
# the objects produced by the map() function have an attribute
# called "coef" that holds the mean estimates for each parameter
print(fit1@coef)
# we access individual estimates by indexing with the parameter name
a_mean = fit1@coef['a']
b_mean = fit1@coef['b']
# It is simple to calculate the mean predicted income for men and women
# (remembering that our model predicts log income)
mean_income_men <- exp(a_mean)
mean_income_women <- exp(a_mean + b_mean)
print(c(men=mean_income_men,women=mean_income_women))

What about the joint posterior? We can tell something about it by examining the variance-covariance matrix of the coefficients. A variance-covariance matrix tells you the variance (standard deviation squared) of each variable in a joint distribution, along with the covariance of each pair of variables. (Covariance is similar to correlation, but it has not been standardized to fit into [-1,1]).

In [6]:
vc <- vcov(fit1)
# rounding to 8 digits for legibility
round(vc,8)
absigma
a 0.01450941-0.01450914-0.00000107
b-0.01450914 0.03154188 0.00000112
sigma-0.00000107 0.00000112 0.00391765

In the above variance-covariance matrix, the elements along the diagonal tell us that $Var(a)=0.01450941$, $Var(b)=0.03154188$, and $Var(\sigma)=0.00391765$ (these are just the standard deviations reported above, squared). But more interestingly, it tells us that a and b have a pretty significant covariance $Cov(a,b)=-0.01450914$. This means that higher values for a tend to go along with lower values for b, and vice versa.

We can look a little more closely at this distribution by taking a sample from it.

We can also work with the samples to create posteriors for parameters we didn't explicitly estimate. It could be useful to compare the posterior distribution for the mean earnings for men (a) with the posterior distribution for the earnings of women (a+b).

In [10]:
posterior_mu_men <- density(samp1$a)
posterior_mu_women <- density(samp1$a + samp1$b)

# get good limits for the plot
xlim <- range(posterior_mu_men$x, posterior_mu_women$x)
ylim <- range(posterior_mu_men$y, posterior_mu_women$y)
plot(posterior_mu_men,xlab='log earnings',xlim=xlim,ylim=ylim,main="")
lines(posterior_mu_women,lty=3)
# add a legend
legend('topleft',lty=c(1,3),legend=c("men","women"))

Adding covariates¶

To make the model more complete, let's add covariates for age and college education.

# construct a binary variable for college education
d$college <- d$education=="4+ years college"
# make a new model
m2 <- alist(
    log_income ~ dnorm(mu,sigma),
    mu <- a + bf*female + ba*age + bc*college,
    a ~ dnorm(0,30),
    bf ~ dnorm(0,30),
    ba ~ dnorm(0,30),
    bc ~ dnorm(0,30),
