Defining functions¶

Sometimes it is useful to define a function in R if you are going to be making the same calculation over and over again. For example, calculating the logit (log odds) of a probability p:

In [7]:
p <- .7
log(p/(1-p))

p <- .1
log(p/(1-p))
0.847297860387203
-2.19722457733622

Functions make it easy to automate repeated tasks like this.

In [8]:
my_logit <- function(p){
    log(p/(1-p))
}

my_logit(.7)
my_logit(.1)
0.847297860387203
-2.19722457733622

This can be very useful for standardizing variables.

In [9]:
st <- function(x){
    (x-mean(x,na.rm=TRUE))/sd(x,na.rm=TRUE)
}

d$grade.s <- st(d$grade)

We use the logit() function as a link, and the map2stan() function to estimate the mode. The syntax for map2stan() is almost identical to map(), but it can estimate a much larger class of models.

(This function will take a little time to run. It is writing and compiling a C++ program to estimate our model for us.)

In [13]:
library(rethinking)
m1 <- alist(
    ever_cocaine ~ dbinom(1,p),
    logit(p) <- a,
    a <- dnorm(0,1.5)
)
f1 <- map2stan(m1,data=d[complete.cases(d$ever_cocaine),])
precis(f1)
