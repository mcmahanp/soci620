Define the model using the dcategorical and softmax functions.

In [7]:
# the order of arguments to dcategorical determines which
# category is which
m <- alist(
    ms ~ dcategorical(softmax(0,s_s,s_d,s_w)),
    s_s <-  a_s + b_s*log_income,
    s_d <-  a_d + b_d*log_income,
    s_w <-  a_w + b_w*log_income,
    c(a_s,b_s,a_d) ~ dnorm(0,2),
    c(b_d,a_w,b_w) ~ dnorm(0,3)
)

Aside from the sign and confidence of the estimates, it is very hard to interpret these coefficients on their own. We can say that income is negatively associated with being single, divorced, or widowed in comparison with being married. That is, as income increases, the model predicts a higher probability of being married rather than single, of being married rather than divorced, and of being married rather than widowed.

We can get a better idea of the specific relationships by predicting outcomes for specific sets of covariates. We'll start by comparing people that make \$10,000 per year to people that make \$100,000 per year.

# calculate posterior median and 90%-interval for probability
# (vapply is like sapply, but you tell it the shape of the
# result of each iteration. This is necessary here because)
# each loop through grid returns a 3x4 matrix)
ppost <- vapply(1:length(grid),function(i){ # for each point on the grid
    # make a matrix of just the s values for this level of income
    ssamp <- cbind(0,post$s_s[,i],post$s_d[,i],post$s_w[,i])
    # use sm to convert to probabilities
    psamp <- apply(ssamp,1,sm)
    # get the 0.05, 0.5, and 0.95 quantile
    # this gives us the 90% interval and the median
    prange <- apply(psamp,1,quantile,probs=c(0.05,0.5,0.95))
},matrix(0.0,nrow=3,ncol=4))

Another useful plot is a 'stacked' probability plot. It is better at showing total proportions, but is worse at showint the uncertainty in our posterior estimates. We can use the same simulated data from the previous plot to make a stacked plot.

# define colors

for(i in 1:4){# four categories
    # get the cumulative probability
    if(i==1){
        # different logic for the first row
        cum_p <- ppost[2,i,]
    }else{
        cum_p <- apply(ppost[2,1:i,],2,sum)
    }
    # draw a polygon from the bottom up to our values
    # note, this will only work with a color that has
    # some transparency. otherwise the final polygon will
    # simply cover the rest.
    polygon(
        c(grid,rev(grid)),
        c(cum_p,rep(0,length(grid))),

}
