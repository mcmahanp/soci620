Covariance in multilevel models¶

This lab will show you two different ways to estimate a multilevel model that has correlated random effects (models in which the between-group variation of multiple coefficients are not assumed independent). First, we will construct a model using the rethinking package, which is very useful to help understand how the model is structured and where the priors come into play. Then, we will estimate a nearly identical model using the brms package. This package uses a special syntax for specifying multilevel models (the same syntax as the popular lme4 package) and is much more computationally efficient than the functions provided in rethinking.

reindex_1 <- function(x){
    # re-index an id x to exactly 1:length(x)
    x_old <- unique(as.character(x))
    x_old <- x_old[!is.na(x_old)]
    x_new <- seq(length(x_old))
    names(x_new) <- x_old
    return(x_new[as.character(x)])
}

The full specification of the model looks like:

$$S_{ik} \sim \mathrm{Norm}(\mu_{ik},\sigma)\\ \mu_{ik} = \beta_{0k} + \beta_{1k}Age_i \\ \\ \beta_{0k} = \gamma_{00} + \eta_{0k}\\ \beta_{1k} = \gamma_{10} + \eta_{1k}\\ \\ \begin{bmatrix}\eta_{0k}\\\eta_{1k}\end{bmatrix} \sim \mathrm{MVNorm}\left(\begin{bmatrix}0\\0\end{bmatrix}, \Phi\right)\\ \\ \Phi = \begin{bmatrix}\phi_0&0\\0&\phi_1\end{bmatrix}R\begin{bmatrix}\phi_0&0\\0&\phi_1\end{bmatrix}\\ \\ \sigma \sim \mathrm{Unif}(0,100)\\ \gamma_{00} \sim \mathrm{Norm}(500,100)\\ \gamma_{10} \sim \mathrm{Norm}(0,20)\\ \phi_0 \sim \mathrm{Unif}(0,100)\\ \phi_1 \sim \mathrm{Unif}(0,100)\\ R \sim \mathrm{LKJ}(2,2)$$

The key difference between this model and the simple random-intercept models we have already seen is the explicit modelling of the covariance between $\eta_{0k}$ (class-specific intercepts) and $\eta_{1k}$ (class-specific slopes), which is captured with the covariance matrix $\Phi$. There are many ways to estimate such covariance matrices, but a very robust method is to decompose it into a vector of standard deviations $(\phi_0,\phi_1)$ and a correlation matrix $R$. Here we use the method of Lewandowski, Kurowicka, and Joe (LKJ; 2009) to set a prior on $R$.

Estimation with rethinking¶

To estimate the model using the rethinking package, we use a specification very similar to the mathematical specification above. The key points to notice in this specification are:

  • Both b_0 and b_age are indexed by class id ([class_id]).
  • We use c(b_0,b_age)[class_id] ~ to denote the joint distribution of b_0 and b_age. This differs from standard R syntax significantly. In R, c(b_0,b_age) would construct a one-dimensional vector with a length twice the number of classes. The model code below should be thought of as a special way to tell map2stan how to build a Stan model.
  • the dlkjcorr() function specifies an LKJ prior on rho with concentration parameter $\zeta=2$. The dimension of the matrix is inferred from the data.

m_rt <- alist(
    reading_score ~ dnorm(mu,sigma),
    mu <- b_0[class_id] + b_age[class_id]*age_s,
    c(b_0,b_age)[class_id] ~ dmvnorm2(c(gamma_00,gamma_10),phi,rho),
    sigma ~ dunif(0,100),
    gamma_00 ~ dnorm(500,100),
    gamma_10 ~ dnorm(0,10),
    phi ~ dunif(0,50),
    rho ~ dlkjcorr(2)
)

Estimation with brms¶

We can estimate an (almost) identical model using the brm() function from the brms package.

The main argument to brm is the model formula. The notation is similar to the standard R notation for linear regressions using lm(), but with extra components to specify grouping. In this case, (1+age_s|teacher_id) tells the moddel to use a random coefficient for the intercept (1) and age (age_s), and to have them grouped by teacher_id. Note that with brm we do not need to deal with re-indexing or removing missing values. All of that happens under the hood.

The other argument to pay attention to is the prior argument. brm has special ways to specify priors, and picks (usually) reasonable default priors for any that are left unspecified.

        set_prior('lkj(2)',class='cor')
    ),
    data=d,chains=4,cores=4,
