We get estimates and confidence intervals for the "population-level" effects, as well as a estimates for sigma. The model also reports two important pieces of diagnostic information.

Eff.Sample: This refers to the "effective" posterior sample size. Autocorrelation in the HMC chain can mean that you need a larger sample to get good estimates for the mean and credible intervals on your parameters. Usually an effective sample of a few hundred is sufficient to get good values for the mean, median, and 95% credible intervals. However if you want wider credible intervals (like 99%, say), you will want a bigger effective sample size.

Rhat: When you run multiple HMC chains to estimate your model (this is very good practice), "Rhat" ($\hat{R}$) is a measure of how much the samples from those chains look like they are from the same posterior. You want $\hat{R}$ to be close to 1.0. If it is bigger than 1.2 or so, you should start to worry that your chains haven't converged and you should specify more iterations and a longer warmup. brm should warn you about this but it is a good idea to check.

By default, brm() uses improper flat priors for the slope coefficients. This is a little bit controversial, but works well enough in practice.

The intercept is treated differently. It is given a t-distributed prior, centered at the mean of the outcome variable. This is also a somewhat problematic choice in principle: basing priors on your data can defeat the purpose of the prior. The standard deviation is quite wide, and the t distribution (with 3 degrees of freedom) has very fat tails, so this won't tend to induce any bias.

Still, let's put some more principled priors on our coefficients.

The summary now has some information on "group-level" (as opposed to "population-level") effects. Because we only have a random intercept the information on this is limited to a standard deviation. The estimate puts it at around 25, which tells us the scale of variation between classes in average test score. 100 points (about four standard deviations) will cover about 95% of the variation in average scores between the classes.

To round out this version of the model, we can add random effects to the non-intercept (slope) coefficients:

$$ S_{ik} \sim \mathrm{Norm}(\mu_{ik},\sigma)\\ \mu_{ik} = \beta_{0k} + \beta_{1k}Age_i + \beta_{2k}Female_i + \beta_{3k}Black_i\\ \beta_{0k} = \gamma_{00} + \eta_{0k}\\ \beta_{1k} = \gamma_{10} + \eta_{1k}\\ \beta_{2k} = \gamma_{20} + \eta_{2k}\\ \beta_{3k} = \gamma_{30} + \eta_{3k} $$

In addition to expanding the model formula, we added a few more arguments this time. The first few (chains, iter, warmup and cores) increase the number of chains from 4 to 8 and try to spread it across 8 different cores on the computer. This maintains the same number of total posterior samples (4000) and same warmup period (1000) as the default. For longer-running models this kind of parallel execution can speed things up considerably if your computer has multiple processing cores. (You can see how many cores R thinks your computer has by typing parallel::detectCores()).

The last argument, control, specifies specific parameters for Stan to use. Setting adapt_delta is a common one, which can help deal with warnings about divergent samples.

Adding second-level (group-level) predictors¶

Adding group-level predictors is straightforward, but a little more complex than what we've done so far. In the slides, we saw how to expand a multilevel model into a "mixed-effects" model with interactions and random components.

We will start by adding predictors just to the intercept:

$$ S_{ik} \sim \mathrm{Norm}(\mu_{ik},\sigma)\\ \mu_{ik} = \beta_{0k} + \beta_{1k}Age_i + \beta_{2k}Female_i + \beta_{3k}Black_i\\ \\ \beta_{0k} = \gamma_{00} + \gamma_{01}Size_k + \gamma_{02}Exp_i + \eta_{0k}\\ \beta_{1k} = \gamma_{10} + \eta_{1k}\\ \beta_{2k} = \gamma_{20} + \eta_{2k}\\ \beta_{3k} = \gamma_{30} + \eta_{3k}\\ $$

Adding predictor variables to the slope parameters means that they have to interact with the population-level variable. (Note: if you are not including the covariates in the intercept, this gets more complicated).

$$ S_{ik} \sim \mathrm{Norm}(\mu_{ik},\sigma)\\ \mu_{ik} = \beta_{0k} + \beta_{1k}Age_i + \beta_{2k}Female_i + \beta_{3k}Black_i\\ \\ \beta_{0k} = \gamma_{00} + \gamma_{01}Size_k + \gamma_{02}Exp_i +\eta_{0k}\\ \beta_{1k} = \gamma_{10} + \gamma_{11}Size_k + \eta_{1k}\\ \beta_{2k} = \gamma_{20} + \eta_{2k}\\ \beta_{3k} = \gamma_{30} + \eta_{3k}\\ $$

Generalized models¶

As a brief example of using brm() to estimate generalized multilevel models (binomial, poisson, multinomial, etc.), we will specify a simple model predicting whether a student does better on their math test than their reading test (a binary outcome):

$$ M_{ik} \sim \mathrm{Binomial}(1,p_{ik})\\ \mathrm{logit}(p_{ik}) = \beta_{0k} + \beta_{1k}Age_i + \beta_{2k}Female_i\\ \\ \beta_{0k} = \gamma_{00} + \gamma_{01}TExp_k + \gamma_{02}TFemale_k \eta_{0k}\\ \beta_{1k} = \gamma_{10} + \eta_{1k}\\ \beta_{2k} = \gamma_{20} + \eta_{2k} $$
