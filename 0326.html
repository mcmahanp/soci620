Random-intercept model¶

Picking up where we left off, we will ask brm() to estimate a random-intercept model. For illustrative purposes, let's set different priors for the intercept ($\gamma_{00}$), the coefficient on age ($\beta_1$}, and use the same prior for coefficients on the two indicator variables ($\beta_2$ and $\beta_3$). We will use half-Cauchy distributions for the standard-deviation paramters.

$$ S_{ik} \sim \mathrm{Norm}(\mu_{ik},\sigma)\\ \mu_{ik} = \beta_{0k} + \beta_{1}Age_i + \beta_{2}Female_i + \beta_{3}Black_i\\ \beta_{0k} = \gamma_{00} + \eta_{0k}\\ \eta_{0k} \sim \mathrm{Norm}(0,\phi_0) $$

$$ \gamma_{00}\sim\mathrm{Norm}(500,100)\\ \beta_1\sim\mathrm{Norm}(0,50)\\ \beta_1,\beta_2\sim\mathrm{Norm}(0,100)\\ \sigma\sim \mathrm{halfCauchy}(0,50)\\ \phi_0\sim \mathrm{halfCauchy}(0,50) $$

In [6]:
# you can break the formula up onto multiple lines, as long as
# each line ends with an operation like `~` or `+`
m_1 <- student_reading_score ~
    1 + student_age_s + student_re_black + student_female +
    (1 | teacher_id)

fit_1 <- brm(
    formula=m_1,
    data=d,
    prior=c(
        set_prior("normal(500,100)",class="Intercept"), # gamma_00
        set_prior("normal(0,100)",class='b'), # default prior for coefficients
        set_prior("normal(0,50)",class='b',coef='student_age_s'), # exception for `student_age_s`
        set_prior("cauchy(0,50)",class='sigma'), # sigma
        set_prior("cauchy(0,50)",class='sd') # phi_0
    )
)

Setting priors in brm() is quite awkward, especially because it does not know about the hierarchical structure we used to construct our model. The function breaks up priors into multiple 'classes' of parameter. The (fixed) intercept always has its own prior, with class "Intercept".

Every coefficient (Any $\beta$ parameter that does not have a random component, and all of the $\gamma$ parameters) has class 'b'. You can set one prior for all of these coefficients, and then set exceptions to that prior for coefficients that you want different priors for. In the code above, we made the default prior on coefficients $\mathrm{Norm}(0,100)$, but explicitly set a narrower prior for student_age_s.

The population-level standard deviation ($\sigma$) has class 'sigma', while the standard deviation the random effects all are of class 'sd'. In this case we only have one second-level random effect ($\phi_0$).

This system for setting priors is very flexible, but not especially intuitive. You can read the sometimes-helpful description of priors in brms with ?set_prior.

We'll make sure our priors were set correctly by asking to look at them with the prior_summary() function:

Random slopes¶

Adding random slopes to the model is straightforward when there are no second-level predictors for the slopes (e.g. the effect of teacher sex on $\beta{2k}$). The model is specified in a hierarchical framework by adding formulas for each of the level-one $\beta$ parameters. We also model the joint distribution of $(\eta_{0k},\eta_{1k},\eta_{2k},\eta_{3k}) = \eta_{\cdot k}$ using a multivariate normal distribution with an LKJ(2,4) prior on the correlation matrix.

$$\begin{align} S_{ik} &\sim \mathrm{Norm}(\mu_{ik},\sigma)\\ \mu_{ik} &= \beta_{0k} + \beta_{1k}Age_i + \beta_{2k}Female_i + \beta_{3k}Black_i\\ \beta_{0k} &= \gamma_{00} + \eta_{0k}\\ \beta_{1k} &= \gamma_{10} + \eta_{1k}\\ \beta_{2k} &= \gamma_{20} + \eta_{2k}\\ \beta_{3k} &= \gamma_{30} + \eta_{3k}\\ \eta_{\cdot k} &\sim \mathrm{MVNorm}(0,\Phi,R)\\ \\ \gamma_{00}&\sim\mathrm{Norm}(500,100)\\ \gamma_{10}&\sim\mathrm{Norm}(0,50)\\ \gamma_{20},\gamma_{30}&\sim\mathrm{Norm}(0,100)\\ \sigma&\sim \mathrm{halfCauchy}(0,50)\\ \phi_k&\sim \mathrm{halfCauchy}(0,50)\\ R &\sim LKJ(2,4) \end{align} $$

In [27]:
m_2 <- student_reading_score ~ 
    student_age_s + student_re_black + student_female +
    (1 + student_age_s + student_re_black + student_female | teacher_id)

fit_2 <- brm(
    formula=m_2,
    data=d,
    prior=c(
        set_prior("normal(500,100)",class="Intercept"), # gamma_00
        set_prior("normal(0,100)",class='b'), # default prior for coefficients
        set_prior("normal(0,50)",class='b',coef='student_age_s'), # exception for `student_age_s`
        set_prior("cauchy(0,50)",class='sigma'), # sigma
        set_prior("cauchy(0,50)",class='sd'), # phi_0, phi_1, phi_2, phi_3,
        set_prior('lkj(2)',class='cor') # R (this can have class='L')
    ),
    chains=8,iter=1500,warmup=1000,cores=8,
    control=list(adapt_delta=.99)
)

We added one new line to prior, which specifies the LKJ prior on the correlation matrix. LKJ is actually the default prior for correlations in brm(), but it uses a concentration parameter of 1 unless you tell it otherwise.

We also snuck in a few more arguments to brm(). The last one, control, is just a way to pass "control" instructions to the underlying Stan program. In this case, telling Stan to set adapt_delta=.99 has it take smaller steps as it is tuning its HMC chain. You don't need to know the details of this, but it is a pretty common adjustment when using Stan.

We also added values for chains, iter, warmup, and cores. These are not strictly necessary, but learning to use them can make your models run much faster on a multi-core computer (like most are these days). The following figure can help explain how this speedup can work. In general, once the Stan program is compiled, it can run each HMC chain in its own 'process' or 'thread' (we can ignore the distinction between 'process' and 'thread' here). Ideally, these processes execute in parallel on your computer, multitasking the job of drawing posterior samples. The figure below illustrates the possible gains from increasing the number of processes that you allow brm() to launch, and the number of HMC chains you tell it to use.

In the first example, there are two chains being run on one thread. By splitting this up into two threads (cores=2, e.g.) in the second example, the sampling portion of the model takes about half as long. In the third example, we get the same number of post-warmup samples by runnign four chains on four threads, but cutting the number of post-warmup iterations in half. (Note that in practice, you almost always want at least 4 chains, which helps with effective sample size and accurate values of $\hat{R}$.)

In the function call above, we split the 4000 post-warmup sample across 8 chains in 8 threads, each of which had 1000 warmup iterations and 1500-1000=500 post-warmup iterations.

You can tell how many cores your computer has with the parallel::detectCores() function.

Group-level predictors¶

For the sake of completenesss, let's estimate the full model from lecture, including group-level predictors (leaving off the priors for brevity).

$$\begin{align} S_{ik} &\sim \mathrm{Norm}(\mu_{ik},\sigma)\\ \mu_{ik} &= \beta_{0k} + \beta_{1k}Age_i + \beta_{2k}Female_i + \beta_{3k}Black_i\\ \\ \beta_{0k} &= \gamma_{00} + \gamma_{01}Size_k + \gamma_{02}Exp_i + \gamma_{03}TeacherFemale_k+\\ &\phantom{{}={}}\gamma_{04}TeacherBlack_k+\gamma_{05}PropBlack_k+\eta_{0k}\\ \beta_{1k} &= \gamma_{10} + \gamma_{11}Size_k + \eta_{1k}\\ \beta_{2k} &= \gamma_{20} + \gamma_{21}TeacherFemale_k + \eta_{2k}\\ \beta_{3k} &= \gamma_{30} + \gamma_{31}TeacherBlack_k + \gamma_{32}PropBlack_k+\eta_{3k} \end{align} $$
